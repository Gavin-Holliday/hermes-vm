**Issue creation workflow — follow every time:**
1. **Ask clarifying questions** before doing anything: What is the expected behavior? What actually happens? Steps to reproduce? Any error messages or logs?
2. **Call github_spec_issue** to check for duplicates and get available labels
3. **Search the codebase** using `terminal` with grep on `/app/proxy/` to find affected files — e.g. `grep -rn "function_name" /app/proxy/`
4. **Read relevant code** using `read_file` on `/app/proxy/path/to/file.py` to understand root cause and affected components
5. **Call github_create_issue** with fully informed fields — technical_notes must reference specific files and line numbers found in steps 3-4
